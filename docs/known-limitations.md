# Known Limitations

- `script` execution is intentionally partial and currently supports only simple commands, quoting, comments, `|`, `<`, `>`, `>>`, `2>`, `2>>`, `2>&1`, `;`, `&&`, `||`, minimal `if ... then ... [elif] ... [else] ... fi`, `while ... do ... done`, command-local assignments, `$NAME` / `${NAME}` / `${NAME:-default}` / `$1` / `$2` / `$@` / `$#` expansion, and argument globbing with `*`, `?`, and bracket classes
- only the virtual safe backend is functional in the bootstrap
- only a narrow set of builtins is implemented: `echo`, `cd`, `export`, `expr`, `time`, `timeout`, `whoami`, `hostname`, `help`, `clear`, `history`, `alias`, `unalias`, `bash`, `sh`, `env`, `which`, `dirname`, `basename`, `curl`, `tree`, `stat`, `du`, `file`, `readlink`, `ln`, `pwd`, `printenv`, `cat`, `grep`, `egrep`, `fgrep`, `wc`, `sort`, `uniq`, `head`, `tail`, `cut`, `tr`, `paste`, `sed`, `join`, `awk`, `jq`, `yq`, `sqlite3`, `find`, `ls`, `rev`, `nl`, `tac`, `strings`, `fold`, `expand`, `unexpand`, `html-to-markdown`, `rm`, `rmdir`, `cp`, `mv`, `comm`, `diff`, `column`, `chmod`, `python`, `python3`, `js-exec`, `xan`, `xargs`, `rg`, `split`, `od`, `base64`, `md5sum`, `sha1sum`, `sha256sum`, `gzip`, `gunzip`, `zcat`, `tar`, `tee`, `printf`, `seq`, `date`, `mkdir`, `touch`, `true`, `false`, `sleep`
- host-backed filesystem support is limited to a single `/workspace` mount
- copy-on-write overlay still does not support deleting host-backed paths or true whiteouts; `rm` in `host_cow` only removes overlay-only entries
- host-side file helpers are embedding APIs and are not intended to replace shell-first agent workflows
- network remains zero-trust by default; `curl` only works with explicit network policy plus per-exec `network_enabled=True`
- detached runs are in-process only and are lost if the embedding interpreter exits
- only one active run is allowed per `Bash` session
- run events and output accessors are buffered snapshots; there is no live streaming log API yet
- pipeline execution is buffered and sequential rather than streaming or process-concurrent
- shell state defaults to session-persistent behavior; `session_state="per_exec"` resets `cwd`, exported env, aliases, and history between calls, but filesystem contents remain shared
- variable support is limited to explicit request env, `export`, and command-local assignments; there is no general persistent shell variable state beyond exported env
- globbing is limited to script arguments; command names, redirection targets, and argv mode stay literal
- control flow is limited to minimal `if ... then ... [elif] ... [else] ... fi` and `while ... do ... done`; `for`, `until`, and `case` are still unsupported
- text builtins are intentionally narrow: `cd` only supports zero or one path and persists session cwd only when explicitly invoked, `export` only supports plain names and `NAME=value`, `expr` only supports one literal value or one binary operator, `time` and `timeout` only wrap one nested command, `whoami` only reads `USER`/`LOGNAME`, `hostname` is fixed, `help` only lists allowlisted commands, `clear` only emits ANSI clear-screen bytes, `history` is a buffered per-session request log, `alias`/`unalias` only support simple first-word expansion, `bash`/`sh` only support `-c <script>` or one script path with isolated child-shell state, `env` only supports `-i`, inline `KEY=VALUE`, and optional command exec, `which` only checks the sandbox allowlist, `dirname`/`basename` are path-string transforms, `curl` only supports one URL plus narrow `-X`, `-d`, `-I`, `-i`, `-o`, and `-L` under explicit policy, `tree` only supports `-a` and `-L`, `stat` only reports narrow type, mode bits, and size-or-entry metadata, `du` only supports byte-based size accumulation plus `-a`, `-h`, `-s`, `-c`, and `--max-depth=N`, `file` only distinguishes symlink/directory/empty/text/data, `readlink` only returns sanitized workspace targets, `ln` only supports one `TARGET LINK_NAME` pair plus narrow `-s`, `-f`, `-n`, and `-v`; memory-mode hard links are copy-like rather than inode-shared, `host_readwrite` uses real host hard links, and `host_cow` still does not support hard-link creation, `html-to-markdown` only supports one file or stdin plus narrow `--bullet`, `--code`, `--hr`, `--heading-style`, and `--help`, `grep` now supports regex search plus `-E`, `-F`, `-i`, `-n`, `-v`, `-c`, `-l`, and `-r`, but it still only handles UTF-8 text, only short flags, treats default regex mode as the current regex engine rather than GNU BRE fidelity, and only prefixes filenames for multi-file or recursive searches, `egrep` is only the current alias to regex `grep`, `fgrep` is only the current alias to fixed-string `grep`, `wc` reports aggregate counts, `sort` is plain lexical line sort, `uniq` only collapses adjacent duplicates, `head`/`tail` only support `-n`, `cut` only supports delimiter-based field selection, `tr` only supports literal transliteration plus `-d` deletion on UTF-8 text, `paste` only supports line-column joins with an optional single-character delimiter, `sed` only supports literal `s/old/new/` substitution with optional `g`, `join` only supports exactly two pre-sorted inputs plus optional `-t`, `-1`, and `-2`, `awk` now supports optional `-F`, `-v`, `BEGIN`/`END`, variables, `print`, basic assignments (`=`, `+=`, `-=`, `*=`, `/=`), arithmetic, `$0/$N`, `NF/NR/FNR/FILENAME`, and basic comparisons / `&&` / `||` / `~`, but it still omits regex literals, `printf`, user functions, arrays, `next`, `getline`, loops, and most of the upstream language surface, `jq` now supports paths, literals, array/object construction, `select`, `map`, `length`, `type`, `keys`, `has`, `//`, comparisons, and basic arithmetic, but it still omits assignment/update semantics, recursion, variables, reducers, regex/string builtin depth, and most of the upstream function surface, `yq` now supports YAML by default, auto-detects `.json`/`.toml`/`.csv`/`.tsv`/`.ini`/`.xml` file inputs, accepts explicit JSON/TOML/CSV/INI/XML input and YAML/JSON/TOML/CSV/INI/XML output, supports `--front-matter` extraction, and inherits the current jq-lite filter surface, but it still omits in-place edits and the broader upstream function surface, `sqlite3` only supports file or `:memory:` databases, SQL from one arg or stdin, and narrow `-json` / `-csv` / `-header` / `-noheader` / `-separator` / `-readonly` / `-cmd` / `-version`, `gzip` only supports one-step gzip transforms plus narrow `-c` / `-d` / `-k` / `-f` / `-S`, `gunzip` is only the current `gzip -d` alias, `zcat` is only the current `gzip -d -c` alias, `tar` only supports create/list/extract with optional `-f`, `-C`, `-O`, and gzip via `-z` or `.tar.gz`/`.tgz`, with safe extraction that blocks absolute and `..` archive members plus unsafe symlink targets, `find` only supports path roots plus `-name`, `-type`, and `-maxdepth`, `ls` only supports immediate listings plus optional `-a` and `-l`, `rev` only reverses lines, `nl` only numbers all lines, `tac` only reverses lines, `strings` only extracts printable ASCII runs with optional `-n`, `fold` only wraps by width with optional `-w`, `expand`/`unexpand` only support narrow tab conversion, `rmdir` only supports empty-directory removal plus optional `-p`, `rm` only supports path deletion plus optional `-f` and `-r`, `cp` only supports plain copies plus optional recursive directory copy with `-r`, `mv` only supports path moves without flags, `comm` only supports exactly two sorted UTF-8 inputs plus optional `-1`, `-2`, and `-3`, `diff` only supports line-oriented UTF-8 file diffs, `column` only supports plain table alignment with optional `-t` and `-s`, `chmod` only supports numeric or symbolic rwx bits plus optional `-R` and `-v`, and in `host_cow` mode those changes stay sandbox-local, `python` is only the current alias to `python3`, `python3` only supports host-runtime `-c`, `-m`, script-file, or stdin execution with workspace snapshot sync; path patching is best-effort and not full interpreter virtualization, `js-exec` only supports host-runtime `-c`, script-file, or stdin execution with workspace snapshot sync plus best-effort `fs` and `cwd` patching, `xan` now supports UTF-8 CSV plus `headers`, `count`, `head`, `tail`, `slice`, `reverse`, `select`, `drop`, `rename`, `enum`, `search`, `sort`, and `filter`; `select`/`drop` support names, indices, `*`, simple header globs, inclusive ranges, and negation, `rename` supports whole-header replacement or `-s` targeted renames, `enum` only prepends a zero-based index column, `search` is literal substring matching with optional `^`/`$` anchors, and `filter` only supports one simple comparison expression, `xargs` only supports whitespace tokenization plus optional `-n`, `rg` only supports literal recursive search with optional `-n`, `-l`, and `-i`, `split` only supports line-count chunking with `-l`, `od` only supports a narrow hex dump, `base64` only supports encode plus `-d`, checksum commands only emit standard lines, `tee` only supports stdin passthrough plus optional `-a`, `printf` only supports `%s`, `%%`, and basic escapes, `seq` only supports integer sequences in 1-, 2-, or 3-argument form, and `date` only supports default local timestamp output plus narrow `+FORMAT` tokens
- file-descriptor support is still narrow; only `2>`, `2>>`, and `2>&1` are implemented
- shell compatibility is intentionally partial and must not be confused with full bash parity
