# Known Limitations

- `script` execution is intentionally partial and currently supports only simple commands, quoting, comments, `|`, `<`, `>`, `>>`, `2>`, `2>>`, `2>&1`, `;`, `&&`, `||`, minimal `if ... then ... [else] ... fi`, command-local assignments, `$NAME` / `${NAME}` expansion, and argument globbing with `*`, `?`, and bracket classes
- only the virtual safe backend is functional in the bootstrap
- only a narrow set of builtins is implemented: `echo`, `cd`, `export`, `expr`, `time`, `timeout`, `whoami`, `hostname`, `help`, `clear`, `history`, `alias`, `unalias`, `bash`, `sh`, `env`, `which`, `dirname`, `basename`, `curl`, `tree`, `stat`, `file`, `readlink`, `ln`, `pwd`, `printenv`, `cat`, `grep`, `wc`, `sort`, `uniq`, `head`, `tail`, `cut`, `tr`, `paste`, `sed`, `join`, `awk`, `jq`, `yq`, `find`, `ls`, `rev`, `nl`, `tac`, `strings`, `fold`, `expand`, `unexpand`, `rm`, `rmdir`, `cp`, `mv`, `comm`, `diff`, `column`, `xargs`, `rg`, `split`, `od`, `base64`, `md5sum`, `sha1sum`, `sha256sum`, `tee`, `printf`, `seq`, `date`, `mkdir`, `touch`, `true`, `false`, `sleep`
- host-backed filesystem support is limited to a single `/workspace` mount
- copy-on-write overlay still does not support deleting host-backed paths or true whiteouts; `rm` in `host_cow` only removes overlay-only entries
- host-side file helpers are embedding APIs and are not intended to replace shell-first agent workflows
- network remains zero-trust by default; `curl` only works with explicit network policy plus per-exec `network_enabled=True`
- detached runs are in-process only and are lost if the embedding interpreter exits
- only one active run is allowed per `Bash` session
- run events and output accessors are buffered snapshots; there is no live streaming log API yet
- pipeline execution is buffered and sequential rather than streaming or process-concurrent
- variable support is limited to explicit request env, `export`, and command-local assignments; there is no general persistent shell variable state beyond exported env
- globbing is limited to script arguments; command names, redirection targets, and argv mode stay literal
- control flow is limited to minimal `if ... then ... [else] ... fi`; loops, `case`, and `elif` are still unsupported
- text builtins are intentionally narrow: `cd` only supports zero or one path and persists session cwd only when explicitly invoked, `export` only supports plain names and `NAME=value`, `expr` only supports one literal value or one binary operator, `time` and `timeout` only wrap one nested command, `whoami` only reads `USER`/`LOGNAME`, `hostname` is fixed, `help` only lists allowlisted commands, `clear` only emits ANSI clear-screen bytes, `history` is a buffered per-session request log, `alias`/`unalias` only support simple first-word expansion, `bash`/`sh` only support `-c <script>` or one script path with isolated child-shell state, `env` only supports `-i`, inline `KEY=VALUE`, and optional command exec, `which` only checks the sandbox allowlist, `dirname`/`basename` are path-string transforms, `curl` only supports one URL plus narrow `-X`, `-d`, `-I`, `-i`, `-o`, and `-L` under explicit policy, `tree` only supports `-a` and `-L`, `stat` only reports narrow type and size-or-entry metadata, `file` only distinguishes symlink/directory/empty/text/data, `readlink` only returns sanitized workspace targets, `ln` only supports `-s` and only in filesystem modes that can create symlinks, `grep` is literal-only, `wc` reports aggregate counts, `sort` is plain lexical line sort, `uniq` only collapses adjacent duplicates, `head`/`tail` only support `-n`, `cut` only supports delimiter-based field selection, `tr` only supports literal transliteration plus `-d` deletion on UTF-8 text, `paste` only supports line-column joins with an optional single-character delimiter, `sed` only supports literal `s/old/new/` substitution with optional `g`, `join` only supports exactly two pre-sorted inputs plus optional `-t`, `-1`, and `-2`, `awk` only supports `print`, optional `-F`, `$0/$N`, `NF/NR/FNR`, and simple `==` / literal `~` filters, `jq` only supports `.`, `.key`, `.[index]`, `.[start:end]`, `.[]`, `|`, `,`, and narrow `-r` / `-c` / `-e` / `-s` / `-n`, `yq` only supports YAML by default plus optional `-p json`, `-o json`, and jq-lite filters from the current `jq` subset, `find` only supports path roots plus `-name`, `-type`, and `-maxdepth`, `ls` only supports immediate listings plus optional `-a` and `-l`, `rev` only reverses lines, `nl` only numbers all lines, `tac` only reverses lines, `strings` only extracts printable ASCII runs with optional `-n`, `fold` only wraps by width with optional `-w`, `expand`/`unexpand` only support narrow tab conversion, `rmdir` only supports empty-directory removal plus optional `-p`, `rm` only supports path deletion plus optional `-f` and `-r`, `cp` only supports plain copies plus optional recursive directory copy with `-r`, `mv` only supports path moves without flags, `comm` only supports exactly two sorted UTF-8 inputs plus optional `-1`, `-2`, and `-3`, `diff` only supports line-oriented UTF-8 file diffs, `column` only supports plain table alignment with optional `-t` and `-s`, `xargs` only supports whitespace tokenization plus optional `-n`, `rg` only supports literal recursive search with optional `-n`, `-l`, and `-i`, `split` only supports line-count chunking with `-l`, `od` only supports a narrow hex dump, `base64` only supports encode plus `-d`, checksum commands only emit standard lines, `tee` only supports stdin passthrough plus optional `-a`, `printf` only supports `%s`, `%%`, and basic escapes, `seq` only supports integer sequences in 1-, 2-, or 3-argument form, and `date` only supports default local timestamp output plus narrow `+FORMAT` tokens
- file-descriptor support is still narrow; only `2>`, `2>>`, and `2>&1` are implemented
- shell compatibility is intentionally partial and must not be confused with full bash parity
